JS Variables

JavaScript Variables can be declared in 4 ways

  • Automatically - not a good practice
  • Using var - it's old method in JS.
  • Using let - This one started from ES6 JS.
  • Using const - This one started from ES6 JS.

Automatically Variable

Without declarining a varible it's automatically taking as a automatically varible.

Variable var

  • var is a global scope variable.
  • var you can use without initialized
  • var can re-declare in same and multiple scope.
  • var can re-assign in same and multiple scope.
  • var not a block scope variable.

Variable let

  • let you can use without initialized
  • let can't re-declare in same scope.
  • let can re-declare in multiple scope
  • let can re-assign in same and multiple scope.
  • let a block scope variable.

Variable const

  • const you can't use without initialized, must you need to declare before use.
  • const can't re-declare in same scope.
  • const can re-declare in multiple scope
  • const can't re-assign in same scope.
  • const a block scope variable.
  • Always declare variables
  • Always use const if the value should not be changed
  • Always use const if the type should not be changed (Arrays and Objects)
  • Only use let if you can't use const
  • Only use var if you MUST support old browsers.

Main Benefits use of Let & Const

Memory Saving :

let and const are block-scoped, ensuring memory is allocated and released per block, reducing unnecessary usage compared to var.

Conflict Fixing:

let and const prevent re-declaration in the same scope (const also prevents reassignment), avoiding variable conflicts and overwrites.

List of Falsy Values in JavaScript

There are exactly 6 falsy values in JavaScript:

  • false – The boolean false.
  • 0 – The number 0 (both positive and negative).
  • " " or ' ' – Empty strings (both single and double quotes).
  • null – Represents the absence of any value.
  • undefined – Represents a variable that has been declared but not assigned a value.
  • NaN – Represents "Not-a-Number," usually the result of an invalid arithmetic operation.

  • Non-empty strings ("hello", "0", etc.)
  • Non-zero numbers (1, -1, 3.14, etc.)

  • Objects ({})

JavaScript Events

1. Mouse Events

  • click: Fired when an element is clicked.
  • dblclick: Fired when an element is double-clicked.
  • mouseover: Triggered when the mouse pointer is over an element.
  • mouseout: Triggered when the mouse pointer leaves an element.
  • mousedown: Fired when a mouse button is pressed on an element.
  • mouseup: Fired when a mouse button is released over an element.
  • mousemove: Triggered when the mouse pointer is moving over an element.

2. Keyboard Events

  • keydown: Triggered when a key is pressed down.
  • keyup: Fired when a key is released.
  • keypress: Triggered when a key is pressed (but is generally less used now).

3. Form Events

  • submit: Fired when a form is submitted.
  • focus: Triggered when an element (like an input field) gains focus.
  • blur: Fired when an element loses focus.
  • change: Triggered when the value of an input element changes.
  • input: Fired when an input value is modified.

4. Window Events

  • load: Triggered when the page is fully loaded.
  • resize: Fired when the window is resized.
  • scroll: Fired when the page is scrolled.
  • unload: Triggered when the page is about to be unloaded or closed.

5. Touch Events (For mobile devices)

  • touchstart: Triggered when a touch is detected.
  • touchmove: Fired when a touch point is moved across the screen.
  • touchend: Fired when a touch is removed from the screen.

6. Drag Events

  • dragstart: Fired when the user starts dragging an element.
  • dragover: Fired when a dragged element is being dragged over a target.
  • drop: Fired when a dragged element is dropped on a target.

Did You Notice?

The two methods, indexOf() and search(), are equal?

The two methods are NOT equal. These are the differences:

  • The search() method cannot take a second start position argument.
  • The indexOf() method cannot take powerful search values (regular expressions).

JavaScript String match()

The match() method returns an array containing the results of matching a string against a string (or a regular expression).

If a regular expression does not include the g modifier (global search), match() will return only the first match in the string.

JavaScript String matchAll()

The matchAll() method returns an iterator containing the results of matching a string against a string (or a regular expression).

JavaScript String includes()

The includes() method returns true if a string contains a specified value.

Otherwise it returns false.
  • includes() is case sensitive.
  • includes() is an ES6 feature.
  • includes() is not supported in Internet Explorer.

JavaScript String endsWith()

The endsWith() method returns true if a string ends with a specified value.

Otherwise it returns false:
  • startsWith(), endsWith() are case sensitive.
  • startsWith(), endsWith() are an ES6 feature.
  • startsWith(), endsWith() are not supported in Internet Explorer.

Associative Arrays

  • Many programming languages support arrays with named indexes.
  • Arrays with named indexes are called associative arrays (or hashes).
  • JavaScript does not support arrays with named indexes.
  • In JavaScript, arrays always use numbered indexes.

The Difference Between Arrays and Objects

  • In JavaScript, arrays use numbered indexes.
  • In JavaScript, objects use named indexes.

Arrays are a special kind of objects, with numbered indexes.

When to Use Arrays. When to use Objects.

  • JavaScript does not support associative arrays.
  • You should use objects when you want the element names to be strings (text).
  • You should use arrays when you want the element names to be numbers.
